1. **Фильтрация истекших назначений**
   - ❌ Было: API возвращал все pending назначения, включая истекшие
   - ✅ Исправлено: Сервер и клиент теперь фильтруют истекшие назначения
   - Файлы: `backend/routes/assignments.js`, `app/.../OrdersViewModel.kt`

2. **Автоматическое обновление заявок**
   - ✅ Добавлено: Polling каждые 30 секунд, когда мастер на смене
   - Файл: `app/.../OrdersViewModel.kt`

3. **Убрана блокировка верификации**
   - ✅ Мастера могут видеть заявки без верификации (но не могут их принимать)

### 🔍 ДИАГНОСТИКА:

#### 1. Проверьте назначения в БД:
```bash
cd backend
node scripts/check-assignments.js
```

#### 2. Проверьте, что API возвращает активные назначения:
```bash
cd backend
node scripts/verify-api-response.js
```

#### 3. Создайте новый тестовый заказ:
```bash
cd backend
node scripts/create-test-order.js
```

### 📱 ЧТО ПРОВЕРИТЬ В ПРИЛОЖЕНИИ:

1. **Мастер должен быть на смене:**
   - Откройте приложение
   - Нажмите "Начать смену" на экране заявок
   - Статус должен быть "На смене"

2. **Проверьте логи приложения:**
   - Откройте Logcat в Android Studio
   - Фильтр: `OrdersViewModel` или `ApiRepository`
   - Ищите сообщения:
     - `🔄 Запрос назначений`
     - `✅ Получено X назначений`
     - `✅ Активных назначений: X`

3. **Проверьте специализацию:**
   - Профиль → Специализация
   - Должны быть выбраны категории техники

### 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ:

#### Проблема 1: Все назначения истекли
**Симптом:** Назначения есть в БД, но не появляются в приложении

**Решение:**
```bash
cd backend
node scripts/create-test-order.js  # Создать новый заказ
```

#### Проблема 2: Мастер не на смене
**Симптом:** Заявки не обновляются автоматически

**Решение:**
- Включите смену в приложении
- Проверьте в БД: `SELECT is_on_shift FROM masters WHERE id = X`

#### Проблема 3: Неправильная специализация
**Симптом:** Заказы создаются, но не назначаются мастеру

**Решение:**
```bash
cd backend
node scripts/update-master-specialization.js  # Обновить специализации
```

#### Проблема 4: Ошибка авторизации
**Симптом:** В логах ошибка 401/403

**Решение:**
- Проверьте, что токен авторизации сохранен в приложении
- Перелогиньтесь в приложении
