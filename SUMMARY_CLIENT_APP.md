## 🎯 Основная идея

### BestApp (Мастера) - существующее
- **Цель**: Получать и выполнять заказы
- **Функции**: Лента заявок, карта, принятие/отклонение, навигация
- **Package**: `com.example.bestapp`
- **Цвета**: Зелёные оттенки

### ClientApp (Клиенты) - новое
- **Цель**: Создавать заказы на ремонт
- **Функции**: Создание заявки, отслеживание, рейтинг мастера
- **Package**: `com.bestapp.client`
- **Цвета**: Синие оттенки

## 📊 Workflow клиента

```
1. Клиент открывает ClientApp
   ↓
2. Регистрируется/Входит
   ↓
3. На главном экране видит свои заказы
   ↓
4. Нажимает "Создать заказ"
   ↓
5. Заполняет форму:
   - Тип техники
   - Бренд, модель
   - Описание проблемы
   - Адрес
   - Желаемое время
   ↓
6. Отправляет заявку
   ↓
7. Backend получает заявку
   ↓
8. Backend запускает поиск мастера
   ↓
9. Мастер назначается (через BestApp)
   ↓
10. Клиент получает уведомление
   ↓
11. Клиент видит информацию о мастере
   ↓
12. Мастер выполняет ремонт
   ↓
13. Заказ завершается
   ↓
14. Клиент оценивает работу мастера
```

## 🔐 API Endpoints (для ClientApp)

### Авторизация
```
POST /api/auth/register
POST /api/auth/login
```

### Заказы
```
POST /api/orders           # Создать заказ
GET  /api/orders           # Получить мои заказы
GET  /api/orders/:id       # Детали заказа
POST /api/orders/:id/rate  # Оценить мастера
PUT  /api/orders/:id/cancel # Отменить заказ
```

### Профиль
```
GET  /api/clients/me       # Мой профиль
PUT  /api/clients/me       # Обновить профиль
```
