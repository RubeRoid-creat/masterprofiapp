# 🔧 ИСПРАВЛЕНИЕ ПРОБЛЕМЫ С НАЗНАЧЕНИЕМ ЗАКАЗОВ МАСТЕРАМ

## ✅ Исправленные проблемы:

1. **Отсутствие таблицы `master_promotions`** - запрос теперь работает без этой таблицы
2. **Отсутствие колонки `subscription_type`** - запрос адаптирован под текущую схему БД
3. **Мастера не на смене** - создан скрипт для автоматической активации

## 📋 Требования для получения заявок мастерами:

1. **Мастер должен быть на смене** (`is_on_shift = 1`)
   - В приложении: кнопка "Начать смену"

2. **Статус мастера должен быть "available"**
   - Автоматически устанавливается при включении смены

3. **Мастер должен иметь специализацию**
   - В приложении: профиль → нажать на специализацию → выбрать категории техники

4. **Мастер должен быть верифицирован** (для просмотра новых заказов)
   - В приложении: профиль → "Верификация" → загрузить документы

5. **Должны быть новые заказы** (`repair_status = 'new'`)

## 🛠️ Скрипты для диагностики и исправления:

### 1. Диагностика проблем:
```bash
cd backend
node scripts/diagnose-orders-assignment.js
```

Показывает:
- Количество новых заказов
- Статус всех мастеров
- Доступных мастеров
- Соответствие типов устройств
- Последние назначения

### 2. Автоматическое исправление настроек мастеров:
```bash
cd backend
node scripts/fix-masters-for-testing.js
```

Этот скрипт:
- ✅ Включает смену у всех мастеров (`is_on_shift = 1`)
- ✅ Устанавливает статус "available"
- ✅ Добавляет специализацию по умолчанию, если её нет

### 3. Создание тестового заказа:
```bash
cd backend
node scripts/create-test-order.js
```

Создает тестовый заказ и автоматически назначает его доступным мастерам.

## 📱 Инструкция для мастеров:

1. **Обновите приложение** до последней версии
2. **Включите смену** (кнопка "Начать смену" в приложении)
3. **Проверьте специализацию:**
   - Откройте профиль
   - Нажмите на блок "Специализация" или кнопку "Изменить" (карандаш)
   - Выберите категории техники, которые вы ремонтируете
   - Нажмите "ОК"
4. **Пройдите верификацию** (опционально, но рекомендуется)

## 🔍 Что делать, если заявки не появляются:

1. Запустите диагностику:
   ```bash
   cd backend
   node scripts/diagnose-orders-assignment.js
   ```

2. Если мастера не на смене, запустите скрипт исправления:
   ```bash
   node scripts/fix-masters-for-testing.js
   ```

3. Если нет новых заказов, создайте тестовый:
   ```bash
   node scripts/create-test-order.js
   ```

4. Проверьте логи сервера на наличие ошибок

## ⚠️ Важные моменты:

- **Специализация должна совпадать с типом устройства в заказе**
  - Например, если заказ на "Холодильник", у мастера должна быть специализация "Холодильник"

- **Типы устройств должны точно совпадать:**
  - ❌ Неправильно: "Микроволновка"
  - ✅ Правильно: "Микроволновая печь"
  - ❌ Неправильно: "Морозильник"
  - ✅ Правильно: "Морозильный ларь"

- **Мастер должен быть в радиусе работы** (если используется геолокация)
