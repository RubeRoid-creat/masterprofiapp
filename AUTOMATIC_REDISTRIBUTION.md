# 🔄 АВТОМАТИЧЕСКОЕ ПЕРЕРАСПРЕДЕЛЕНИЕ ИСТЕКШИХ НАЗНАЧЕНИЙ

## 🎯 ФУНКЦИОНАЛ

Когда мастер не отвечает на заявку в течение отведенного времени (5 минут), система автоматически:
1. ✅ Обновляет статус назначения на `expired`
2. ✅ Ищет следующего подходящего мастера
3. ✅ Создает новое назначение для следующего мастера
4. ✅ Отправляет уведомление новому мастеру

## ⚙️ КАК ЭТО РАБОТАЕТ

### 1. Таймеры для каждого назначения

При создании назначения устанавливается таймер:
```javascript
const timeout = calculateTimeout(attemptNumber); // 5 минут для первой попытки
const timer = setTimeout(() => {
  handleAssignmentExpiration(assignmentId, orderId);
}, timeout);
```

### 2. Обработка истечения времени

Функция `handleAssignmentExpiration`:
- Проверяет статус назначения
- Обновляет статус на `expired`
- Вызывает `findNextMaster()` для поиска следующего мастера

### 3. Поиск следующего мастера

Функция `findNextMaster`:
- Исключает мастеров, которым уже отправляли этот заказ
- Находит подходящих мастеров (специализация, на смене, доступен)
- Создает новое назначение с увеличенным номером попытки (`attempt_number`)

### 4. Фоновая проверка (защита от сбоев)

Так как таймеры теряются при перезапуске сервера, добавлена фоновая задача:
- ✅ Проверка при запуске сервера
- ✅ Периодическая проверка каждую минуту
- ✅ Обрабатывает все истекшие назначения, которые могли быть пропущены

## 📋 ЛОГИКА ПЕРЕРАСПРЕДЕЛЕНИЯ

### Условия для перераспределения:

1. **Назначение истекло:**
   - `status = 'pending'`
   - `expires_at < now()`

2. **Заказ еще новый:**
   - `repair_status = 'new'`

3. **Есть доступные мастера:**
   - Мастер на смене (`is_on_shift = 1`)
   - Мастер доступен (`status = 'available'`)
   - Специализация соответствует типу устройства
   - Мастер еще не получал этот заказ

### Количество попыток:

- Максимум **5 попыток** на заказ
- После 5 неудачных попыток заказ автоматически отменяется (`status = 'cancelled'`)
- Номер попытки сохраняется в поле `attempt_number`

## ✅ ПРЕИМУЩЕСТВА

1. **Надежность:**
   - Двойная защита: таймеры + фоновая проверка
   - Работает даже после перезапуска сервера

2. **Автоматизация:**
   - Не требует ручного вмешательства
   - Заказы не теряются

3. **Справедливость:**
   - Каждый мастер получает шанс
   - Учитывается рейтинг и загрузка

4. **Контроль:**
   - История всех попыток
   - Ограничение на количество попыток

## 🐛 РЕШЕНИЕ ПРОБЛЕМ

### Проблема: Назначения не перераспределяются

**Проверьте:**
1. ✅ Фоновая задача запущена (см. логи при старте сервера)
2. ✅ Есть доступные мастера (не все уведомлены)
3. ✅ Заказ не отменен (`repair_status = 'new'`)

### Проблема: Заказ отменяется слишком быстро

**Решение:**
- Увеличьте `MAX_ATTEMPTS` в `findNextMaster()`
- Увеличьте таймаут в `calculateTimeout()`

### Проблема: Один мастер получает заказ несколько раз

**Решение:**
- Проверьте, что `excludedMasterIds` правильно формируется
- Убедитесь, что `master_id` корректно сохраняется
