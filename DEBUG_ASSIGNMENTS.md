## 🔍 ЧТО ПРОВЕРИТЬ В ПРИЛОЖЕНИИ

### 1. Проверьте логи (Logcat)

Откройте Android Studio → Logcat, фильтр: `OrdersViewModel` или `ApiRepository`

**Ищите сообщения:**
```
🔄 Запрос назначений
📥 Ответ API: code=200
✅ Получено X назначений
✅ Активных назначений: X
```

**Если видите ошибки:**
```
❌ NO AUTH TOKEN! - токен не загружен, нужно перелогиниться
❌ API вернул ошибку: 401/403 - проблема с авторизацией
❌ Ошибка получения назначений: code=XXX - проблема с запросом
```

### 2. Проверьте, что мастер на смене

- Откройте приложение
- Перейдите на экран заявок
- Включите смену (если не включена)

### 3. Проверьте авторизацию

- Убедитесь, что вы залогинены в приложении
- Проверьте токен в логах: должно быть `✅ Token saved` или `✅ Adding Authorization header`

### 4. Принудительное обновление

- Потяните экран заявок вниз (swipe to refresh)
- Или закройте и откройте приложение заново

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### Проблема 1: Токен не загружается
**Решение:**
- Выйдите из приложения
- Войдите заново
- Проверьте логи: должно быть `✅ Token saved`

### Проблема 2: Мастер не на смене
**Решение:**
- Включите смену в приложении
- Проверьте в БД: `SELECT is_on_shift FROM masters WHERE id = 2`

### Проблема 3: Все назначения истекли
**Решение:**
```bash
cd backend
node scripts/create-test-order.js  # Создать новый заказ
```

### Проблема 4: Неправильная специализация
**Решение:**
- Проверьте специализацию мастера в профиле
- Убедитесь, что выбрана соответствующая категория техники
