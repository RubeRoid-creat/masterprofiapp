## 🔍 Найденные проблемы и исправления:

### 1. **Требование верификации блокировало просмотр заявок**
   - ❌ Было: Мастера не могли видеть заявки без верификации
   - ✅ Исправлено: Верификация требуется только для **принятия** заказов, просмотр доступен всем
   - Файлы: `backend/routes/assignments.js`, `backend/routes/orders.js`

### 2. **Отсутствие автоматического обновления заявок**
   - ❌ Было: Заявки обновлялись только при открытии/закрытии приложения
   - ✅ Исправлено: Добавлено автоматическое обновление каждые **30 секунд**, когда мастер на смене
   - Файл: `app/src/main/java/com/example/bestapp/ui/orders/OrdersViewModel.kt`

### 3. **Проблемы с запросом к БД (отсутствующие таблицы/колонки)**
   - ❌ Было: Запрос падал из-за несуществующих таблиц `master_promotions` и колонки `subscription_type`
   - ✅ Исправлено: Запрос адаптирован для работы без этих зависимостей
   - Файл: `backend/services/assignment-service.js`

### 4. **Мастера не на смене**
   - ❌ Было: Мастера не включили смену в приложении
   - ✅ Исправлено: Создан скрипт автоматической активации смены
   - Скрипт: `backend/scripts/fix-masters-for-testing.js`

### 5. **Отсутствие специализации у мастеров**
   - ❌ Было: У мастера #2 не было специализации
   - ✅ Исправлено: Автоматически установлена специализация по умолчанию
   - Скрипт: `backend/scripts/update-master-specialization.js`

### 6. **Несоответствие названий специализаций**
   - ❌ Было: Неправильные названия ("Микроволновка" вместо "Микроволновая печь")
   - ✅ Исправлено: Список специализаций в приложении обновлен и синхронизирован с сервером
   - Файл: `app/src/main/java/com/example/bestapp/ui/profile/ProfileFragment.kt`

## 🔧 Как получить заявки в приложении:

### **Для мастера:**

1. **Обновите приложение** до последней версии

2. **Включите смену:**
   - Откройте приложение
   - Нажмите кнопку "Начать смену" на экране заявок

3. **Проверьте специализацию:**
   - Перейдите в Профиль
   - Нажмите на блок "Специализация" или кнопку "Изменить" (карандаш)
   - Выберите категории техники, которые вы ремонтируете
   - Сохраните изменения

4. **Заявки появятся автоматически:**
   - При открытии экрана заявок (мгновенно)
   - Каждые 30 секунд, когда мастер на смене (автоматическое обновление)
   - При возврате на экран заявок

## 📱 Что делать, если заявки не появляются:

### 1. **Проверьте логи приложения:**
   - Откройте Logcat в Android Studio
   - Фильтр: `OrdersViewModel` или `ApiRepository`
   - Ищите сообщения с префиксом `🔄`, `✅`, `❌`

### 2. **Проверьте на сервере:**
   ```bash
   cd backend
   node scripts/check-assignments.js
   ```
   Покажет все назначения и их статусы

### 3. **Убедитесь, что:**
   - ✅ Мастер на смене (`is_on_shift = 1`)
   - ✅ Статус мастера `available`
   - ✅ Есть специализация, соответствующая типу устройства в заказе
   - ✅ Есть активные назначения со статусом `pending`

### 4. **Создайте тестовый заказ:**
   ```bash
   cd backend
   node scripts/create-test-order.js
   ```

## 🔄 Автоматическое обновление:

Приложение теперь автоматически запрашивает новые заявки:
- **Каждые 30 секунд**, когда мастер на смене
- При открытии экрана заявок
- При возврате на экран (onResume)

Это гарантирует, что даже без WebSocket заявки появятся максимум через 30 секунд.
