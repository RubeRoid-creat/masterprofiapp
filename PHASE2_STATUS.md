## 2.1. Умное создание заявки (ClientApp)

### ✅ Категории и подкатегории услуг
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: `routes/services.js` - API endpoints для категорий
- Таблица `service_categories` в БД с иерархией (parent_id)
- ClientApp: `CategorySelector.kt` - UI компонент для выбора категорий
- Поддержка подкатегорий и отображение в виде диалогов
- API: `GET /api/services/categories`

### ✅ Шаблоны услуг с фиксированными ценами
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: `routes/services.js` - API endpoints для шаблонов
- Таблица `service_templates` в БД (название, описание, цена, категория)
- ClientApp: `CategorySelector.kt` - отображение шаблонов услуг
- Фильтрация шаблонов по категории
- API: `GET /api/services/templates`

### ✅ Загрузка фото/видео проблемы
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: `routes/orders.js` - endpoints:
  - `POST /api/orders/:id/media/upload` - загрузка файлов через Multer
  - `POST /api/orders/:id/media` - добавление ссылок на медиа
- Middleware: `middleware/upload.js` - обработка файлов через Multer
- Таблица `order_media` в БД для хранения медиафайлов
- ClientApp: `MediaUploader.kt` - компонент для загрузки медиа
- Поддержка камеры и галереи
- Превью загруженных файлов
- Ограничение размера файлов (до 5 файлов за раз)
- API: `POST /api/orders/:id/media/upload`, `POST /api/orders/:id/media`

### ✅ Улучшенная геолокация
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: `services/geocoding-service.js` - геокодирование адресов через Yandex Geocoder API
- Автоматическое определение адреса по координатам (reverse geocoding)
- ClientApp: `MapAddressPicker.kt` - выбор адреса на карте Yandex MapKit
- `LocationUtils.kt` - утилиты для работы с геолокацией
- Автоматическое определение текущего местоположения
- Отображение выбранного адреса
- API: Интегрировано в создание заказа

## 2.2. Прозрачность и доверие

### ✅ Профили мастеров с расширенной информацией
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: 
  - Таблица `master_portfolio` для портфолио работ
  - Таблица `master_certificates` для сертификатов
  - Расширенные поля в таблице `masters` (photo_url, bio, experience_years)
  - API: `GET /api/masters/:id/profile` - получение полного профиля
- ClientApp: 
  - `MasterProfileScreen.kt` - экран профиля мастера
  - Отображение фото мастера, портфолио, сертификатов
  - Отображение специализации, опыта, рейтинга
  - Компоненты: `PortfolioItemCard`, `CertificateCard`

### ✅ Система рейтингов и отзывов
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: 
  - Таблица `reviews` в БД
  - API endpoints в `routes/reviews.js`:
    - `POST /api/reviews` - создание отзыва
    - `GET /api/reviews/master/:masterId` - отзывы мастера
    - `PUT /api/reviews/:id` - обновление отзыва
    - Автоматический пересчет рейтинга мастера
- ClientApp:
  - `ReviewViewModel.kt` - управление состоянием отзывов
  - `ReviewDialog.kt` - диалог создания/редактирования отзыва
  - `RatingBar.kt` - компонент выбора рейтинга (1-5 звезд)
  - `ReviewsSection.kt` - секция отзывов в профиле мастера
  - `ReviewCard.kt` - карточка отзыва
  - Интеграция в `OrderDetailsScreen.kt` - возможность оставить отзыв после завершения заказа
  - Отображение рейтинга в `MasterProfileScreen.kt`

### ✅ Детальный статус заявки в реальном времени
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend:
  - Таблица `order_status_history` для истории статусов
  - WebSocket уведомления о смене статуса
  - Push-уведомления на каждом этапе (через FCM)
  - Статусы: "Поиск мастера" → "Мастер принял" → "В пути" → "Выполняет работу" → "Завершено"
- ClientApp:
  - `OrderStatusTracker.kt` - компонент отслеживания статуса с прогресс-баром
  - Визуализация этапов заказа
  - Отображение истории изменений статуса
  - Интеграция в `OrderDetailsScreen.kt`
  - WebSocket обновления в реальном времени
  - Push-уведомления о смене статуса
