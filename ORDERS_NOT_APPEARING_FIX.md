## 📱 Почему заявки могут не появляться:

### **Основные причины:**

1. **Мастер не подключен к WebSocket**
   - Если приложение закрыто или нет интернета - заявка не придет мгновенно
   - ⚠️ **РЕШЕНИЕ:** Заявка появится при следующем запросе через API (`/api/assignments/my`)

2. **Мастер не на смене**
   - Проверить: `is_on_shift = 1` и `status = 'available'`
   - ⚠️ **РЕШЕНИЕ:** Включить смену в приложении или запустить:
     ```bash
     node scripts/fix-masters-for-testing.js
     ```

3. **Мастер не имеет специализации**
   - Специализация должна совпадать с типом устройства в заказе
   - ⚠️ **РЕШЕНИЕ:** Установить специализацию в профиле

4. **Назначение истекло**
   - Pending назначения имеют таймаут 5-15 минут
   - ⚠️ **РЕШЕНИЕ:** Создать новый заказ или дождаться автоматического переназначения

## 📋 API эндпоинты для получения заявок:

1. **Получить все назначения мастера:**
   ```
   GET /api/assignments/my?status=pending
   ```
   - Теперь работает **БЕЗ верификации**
   - Возвращает все pending назначения

2. **Получить активное назначение для заказа:**
   ```
   GET /api/assignments/order/:orderId/active
   ```
   - Автоматически создаст назначение, если мастер подходит

## ⚠️ Важно:

1. **WebSocket уведомления работают только когда:**
   - Приложение открыто
   - Мастер подключен к интернету
   - Есть активное WebSocket соединение

2. **Если WebSocket не работает:**
   - Заявки все равно доступны через API
   - Приложение должно периодически запрашивать `/api/assignments/my`
   - При открытии приложения заявки загрузятся автоматически

3. **Для реального времени:**
   - Нужно убедиться, что приложение поддерживает WebSocket
   - Мастер должен быть авторизован и подключен
