# Улучшения обработки ошибок загрузки сервера

## Что было сделано:

### 1. Добавлена функция `getErrorMessage()` в `ApiRepository`
   - Определяет тип сетевой ошибки (таймаут, недоступность сервера, проблемы подключения)
   - Возвращает понятные сообщения для пользователя на русском языке
   - Обрабатывает следующие типы ошибок:
     - `SocketTimeoutException` - "Превышено время ожидания ответа сервера"
     - `ConnectException` - "Не удалось подключиться к серверу"
     - `UnknownHostException` - "Сервер недоступен"
     - `IOException` - различные сетевые ошибки

### 2. Улучшена обработка ошибок в методах:
   - `login()` - авторизация
   - `getOrders()` - получение заказов
   - `getOrder()` - получение одного заказа
   - `completeOrder()` - завершение заказа
   - `createOrder()` - создание заказа
   - `acceptAssignmentsBatch()` - массовое принятие заказов
   - `getRejectedAssignments()` - история отклонений
   - `createOrUpdateSchedule()` - создание/обновление расписания
   - `deleteSchedule()` - удаление расписания
   - `getWallet()` - получение кошелька
   - `getMasterStats()` - получение статистики

### 3. Добавлена обработка HTTP кодов ошибок:
   - 404 - "Данные не найдены"
   - 401 - "Требуется авторизация. Войдите заново."
   - 403 - "Доступ запрещен"
   - 500 - "Ошибка сервера. Попробуйте позже."
   - 503 - "Сервер временно недоступен. Попробуйте позже."

## Результат:

Теперь при ошибках загрузки данных с сервера пользователь увидит понятные сообщения вместо технических ошибок:

- ❌ Было: "java.net.SocketTimeoutException: timeout"
- ✅ Стало: "Превышено время ожидания ответа сервера. Проверьте подключение к интернету."

- ❌ Было: "java.net.ConnectException: Connection refused"
- ✅ Стало: "Не удалось подключиться к серверу. Проверьте подключение к интернету."
