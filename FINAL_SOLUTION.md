## 🔍 НАЙДЕННЫЕ ПРИЧИНЫ:

### 1. ❌ Истекшие назначения не фильтровались

   - Приложение не проверяло срок действия назначений
   - **Результат:** Заявки не появлялись, так как все были истекшими

### 2. ❌ Нет автоматического обновления
   - Заявки обновлялись только при открытии экрана
   - Не было периодического polling
   - **Результат:** Новые заявки не появлялись автоматически

### 3. ❌ Блокировка верификацией
   - Неверифицированные мастера не могли видеть заявки
   - **Результат:** Заявки не отображались даже если были активны

## ✅ ВНЕСЕННЫЕ ИСПРАВЛЕНИЯ:

### 1. Фильтрация истекших назначений (СЕРВЕР)
**Файл:** `backend/routes/assignments.js`

```javascript
// Фильтруем истекшие pending назначения на сервере
const now = new Date();
const filteredAssignments = assignments
  .filter(assignment => {
    if (assignment.status === 'pending' && assignment.expires_at) {
      const expiresAt = new Date(assignment.expires_at);
      if (expiresAt < now) {
        return false; // Пропускаем истекшие
      }
    }
    return true;
  });
```

**Результат:** API возвращает только активные назначения.

### 3. Автоматическое обновление заявок
**Файл:** `app/src/main/java/com/example/bestapp/ui/orders/OrdersViewModel.kt`

```kotlin
// Polling каждые 30 секунд, когда мастер на смене
private fun startPollingAssignments() {
    pollingJob = viewModelScope.launch {
        while (isActive && _isShiftActive.value) {
            delay(30000) // 30 секунд
            if (_isShiftActive.value) {
                loadNewOrders()
            }
        }
    }
}
```

**Результат:** Заявки обновляются автоматически каждые 30 секунд.
