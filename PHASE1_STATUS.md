## 1.2. Улучшение Backend для клиентов

### ✅ API endpoint для получения заказов клиента
**Статус:** ✅ **РЕАЛИЗОВАНО** (частично)
- `GET /api/orders` - получает заказы клиента при `role === 'client'`
- Фильтрация по `client_id` автоматически применяется
- **Примечание:** Отдельного endpoint `/api/orders/my` нет, но функциональность реализована через основной endpoint

### ⚠️ API endpoint для получения истории заказов
**Статус:** ⚠️ **ЧАСТИЧНО РЕАЛИЗОВАНО**
- `GET /api/orders/:id/status-history` - история статусов конкретного заказа
- **Отсутствует:** общий endpoint для получения истории всех заказов клиента
- Можно получить через `GET /api/orders` с фильтром по статусу

### ✅ WebSocket события для клиентов
**Статус:** ✅ **РЕАЛИЗОВАНО**
- Backend: `websocket.js` с функцией `broadcastToClient(userId, data)`
- События отправляются при изменении статуса заказа
- События отправляются при назначении мастера
- Интеграция в `routes/orders.js` и `routes/assignments.js`

### ✅ API endpoint для получения профиля мастера
**Статус:** ✅ **РЕАЛИЗОВАНО**
- `GET /api/masters/:id` - получение профиля мастера
- Включает: фото, биографию, опыт, портфолио, сертификаты, рейтинг
- ClientApp: `MasterProfileScreen.kt` для отображения профиля
