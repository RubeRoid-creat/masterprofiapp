# 🔧 Инструкция по исправлению ошибки "no such column: inn"

## ✅ Решение 1: Автоматическое исправление (рекомендуется)

Сервер теперь автоматически добавляет поле `inn` при старте. Просто **перезапустите сервер**:

```bash
# На сервере
cd /path/to/backend
pm2 restart all
# или
npm restart
```

После перезапуска поле `inn` будет автоматически добавлено, и ошибка исчезнет.

## ✅ Решение 2: Ручное применение миграции

Если нужно применить миграцию вручную:

```bash
# На сервере
cd /path/to/backend
npm run migrate-inn
```

Или используйте расширенную проверку:

```bash
npm run ensure-inn
```

## ✅ Решение 3: Прямое выполнение SQL

Если миграция не работает, выполните SQL напрямую:

```bash
# Подключитесь к базе данных SQLite
sqlite3 database.sqlite

# Выполните команду
ALTER TABLE masters ADD COLUMN inn TEXT;

# Выйдите
.quit
```

## 📋 Что было исправлено:

1. ✅ Автоматическое добавление поля `inn` при ошибке "no such column: inn"
2. ✅ Автоматическая проверка при старте сервера
3. ✅ Улучшенная обработка ошибок в методах query.run(), query.get(), query.all()
4. ✅ Скрипт миграции `migrate-inn-to-masters.js`
5. ✅ Скрипт проверки `ensure-inn-column.js`

## 🚀 После применения миграции

Ошибка "no such column: inn" больше не должна появляться. Если ошибка все еще возникает:

1. Проверьте логи сервера при старте - должно быть сообщение "✅ Поле inn присутствует в таблице masters"
2. Запустите `npm run ensure-inn` для детальной проверки
3. Убедитесь, что используется правильная база данных (проверьте DATABASE_PATH в .env)
