# 🔄 Инструкция по обновлению сервера

## Быстрое обновление (Backend + Admin Panel)

### Вариант 1: Через скрипт (рекомендуется)

```bash
# 1. Подключитесь к серверу
ssh root@212.74.227.208

# 2. Перейдите в папку проекта
cd ~/masterprofiapp/backend

# 3. Запустите скрипт обновления
chmod +x update-server.sh
./update-server.sh
```

### Вариант 2: Вручную

```bash
# 1. Подключитесь к серверу
ssh root@212.74.227.208

# 2. Перейдите в папку backend
cd ~/masterprofiapp/backend

# 3. Остановите сервер
pm2 stop bestapp-backend
# или если процесс называется иначе:
pm2 stop all

# 4. Получите обновления из GitHub
git pull origin main

# 5. Установите новые зависимости (если есть)
npm install --production

# 6. Перезапустите сервер
pm2 restart bestapp-backend
# или
pm2 restart all

# 7. Проверьте статус
pm2 status
pm2 logs bestapp-backend --lines 50
```

## Обновление админ-панели

Админ-панель - это React приложение. Если она развернута отдельно:

```bash
# 1. Перейдите в папку админ-панели
cd ~/masterprofiapp/backend/admin-panel

# 2. Получите обновления (если репозиторий в корне проекта, они уже получены)
# git pull уже выполнен выше

# 3. Установите зависимости (если есть новые)
npm install

# 4. Соберите приложение для production
npm run build

# 5. Если админ-панель развернута через Nginx или другой веб-сервер,
#    скопируйте содержимое папки dist в директорию веб-сервера
# Например:
# cp -r dist/* /var/www/admin-panel/
```

## Если админ-панель встроена в backend

Если админ-панель обслуживается через Express (статические файлы):

```bash
# После обновления backend и перезапуска PM2, админ-панель обновится автоматически
# Убедитесь, что в server.js настроена раздача статических файлов из admin-panel/dist
```

## Проверка после обновления

```bash
# 1. Проверьте статус PM2
pm2 status

# 2. Проверьте логи на ошибки
pm2 logs bestapp-backend --lines 100

# 3. Проверьте API
curl http://localhost:3000/api/admin/stats

# 4. Проверьте админ-панель в браузере
# http://212.74.227.208:3000/admin (или ваш домен)
```

## Откат изменений (если что-то пошло не так)

```bash
# 1. Перейдите в папку проекта
cd ~/masterprofiapp/backend

# 2. Посмотрите историю коммитов
git log --oneline -10

# 3. Откатитесь на предыдущий коммит
git reset --hard <commit-hash>

# 4. Перезапустите сервер
pm2 restart bestapp-backend
```

## Автоматическое обновление (опционально)

Можно настроить автоматическое обновление через GitHub Actions или webhook.

## Важные замечания

1. **Не удаляйте .env файл** - он содержит важные настройки
2. **Проверьте логи** после обновления на наличие ошибок
3. **Сделайте бэкап БД** перед обновлением (если есть важные данные):
   ```bash
   npm run backup
   ```
