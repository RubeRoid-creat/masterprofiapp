# Отладка проблемы с заказами

## Проблема
Заказы создаются на сайте (orderId: 4), но не появляются в админ-панели.

## Проверка

### 1. Проверьте логи сервера Next.js

При создании заказа должны быть логи:
- `Sending order to admin panel:` - отправка началась
- `Admin API Response Status:` - статус ответа
- `Admin API Response:` - ответ сервера

**Если видите ошибку:**
- `ADMIN_USER_TOKEN не настроен` - токен не читается из .env
- `401 Unauthorized` - токен неверный или истек
- `403 Forbidden` - пользователь не имеет прав
- `400 Bad Request` - неверный формат данных

### 2. Проверьте .env файл

Убедитесь, что:
- Файл называется `.env` (не `.env.local` или другой)
- Токен добавлен без кавычек или в кавычках:
  ```env
  ADMIN_USER_TOKEN=ваш_токен
  ```
  или
  ```env
  ADMIN_USER_TOKEN="ваш_токен"
  ```
- Нет лишних пробелов
- Сервер перезапущен после добавления токена

### 3. Проверьте переменные окружения

В Next.js переменные без префикса `NEXT_PUBLIC_` доступны только на сервере.
`ADMIN_USER_TOKEN` должен быть без префикса.

### 4. Проверьте доступность backend

Убедитесь, что backend доступен:
```powershell
Invoke-WebRequest -Uri "http://212.74.227.208:3000/api/auth/login" -Method POST
```

### 5. Проверьте токен

Попробуйте использовать токен напрямую:
```powershell
$token = "ваш_токен"
$headers = @{
    "Authorization" = "Bearer $token"
    "Content-Type" = "application/json"
}
$body = '{"device_type":"Холодильник","problem_description":"Тест","address":"Тест","latitude":55.751244,"longitude":37.618423}'
Invoke-RestMethod -Uri "http://212.74.227.208:3000/api/orders" -Method POST -Headers $headers -Body $body
```

## Решение

### Если токен не читается

1. Убедитесь, что файл `.env` в корне проекта
2. Перезапустите сервер полностью (остановите и запустите заново)
3. Проверьте, что нет других `.env` файлов, которые могут переопределять переменные

### Если ошибка 401/403

1. Получите новый токен через админ-панель
2. Обновите `.env`
3. Перезапустите сервер

### Если ошибка 400

Проверьте формат данных в логах сервера.
