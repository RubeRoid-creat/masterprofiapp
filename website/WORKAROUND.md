# Обходное решение - Работа без Prisma

## Проблема
Prisma не может скачать движки из-за сетевых проблем.

## Решение: Работа без Prisma (для тестирования UI)

Если нужно протестировать интерфейс сайта, можно временно работать без Prisma:

### Шаг 1: Установка без Prisma
```powershell
.\setup-without-prisma.ps1
```

Или вручную:
```powershell
npm install --legacy-peer-deps --ignore-scripts
```

### Шаг 2: Запуск проекта
```powershell
npm run dev
```

### Что будет работать:
- ✅ Все страницы сайта
- ✅ Интерфейс и дизайн
- ✅ Навигация
- ✅ Формы (но данные не сохранятся)

### Что НЕ будет работать:
- ❌ API endpoints (будут возвращать ошибки)
- ❌ Сохранение заказов
- ❌ Форум
- ❌ Новости из БД

## Полное решение: Docker

Самый надежный способ - использовать Docker:

```powershell
docker-compose up -d
```

Docker скачает все зависимости в изолированной среде и обойдет сетевые проблемы.

## Альтернатива: Ручная установка движков

1. Перейдите на https://github.com/prisma/prisma-engines/releases
2. Найдите версию для Windows (например, `prisma-query-engine-windows.exe`)
3. Скачайте и распакуйте в `node_modules/.prisma/client/`
4. Запустите `npx prisma generate` снова

## Временное решение для разработки

Можно использовать mock-версии API endpoints. Пример для заказов находится в `app/api/orders/route-mock.ts.example`.

Для использования:
1. Переименуйте `route.ts` в `route.ts.backup`
2. Переименуйте `route-mock.ts.example` в `route.ts`
3. Данные будут храниться в памяти (пропадут при перезапуске)

Это позволит полноценно тестировать интерфейс без БД.
