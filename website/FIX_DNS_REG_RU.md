# Исправление DNS записей для ispravleno.pro на reg.ru

## Проблема

При открытии `ispravleno.pro` показывается ошибка "Сайт размещен некорректно" от reg.ru, потому что DNS записи домена указывают на серверы reg.ru, а не на ваш сервер `212.74.227.208`.

## Решение: Изменить DNS записи в панели управления reg.ru

### Шаг 1: Войдите в панель управления reg.ru

1. Откройте https://www.reg.ru
2. Войдите в личный кабинет
3. Перейдите в раздел "Домены"
4. Найдите домен `ispravleno.pro`
5. Нажмите на домен или кнопку "Управление"

### Шаг 2: Настройте DNS записи

Перейдите в раздел "DNS-серверы и зона" или "Управление DNS".

#### Вариант A: Использовать DNS серверы reg.ru (рекомендуется для доменов .pro)

В разделе "DNS-серверы" убедитесь, что используются DNS серверы reg.ru:
- `ns1.reg.ru`
- `ns2.reg.ru`

Затем в разделе "Записи зоны" или "A-записи" настройте следующие записи:

**Удалите существующие A-записи**, которые указывают на серверы reg.ru, и создайте новые:

1. **A-запись для корневого домена:**
   ```
   Тип: A
   Имя: @ (или оставить пустым)
   Значение (IP-адрес): 212.74.227.208
   TTL: 3600 (или Авто)
   ```

2. **A-запись для www:**
   ```
   Тип: A
   Имя: www
   Значение (IP-адрес): 212.74.227.208
   TTL: 3600 (или Авто)
   ```

#### Вариант B: Использовать собственные DNS серверы (если есть)

Если у вас есть собственные DNS серверы, измените DNS серверы домена на ваши.

### Шаг 3: Сохраните изменения

Нажмите кнопку "Сохранить" или "Применить изменения".

### Шаг 4: Ожидание распространения DNS

Изменения DNS могут распространяться от 5 минут до 48 часов. Обычно это занимает 10-30 минут.

Проверить можно командой:
```bash
dig ispravleno.pro
# или
nslookup ispravleno.pro
```

Должно вернуться: `212.74.227.208`

### Шаг 5: После обновления DNS

После того, как DNS записи обновятся, можно продолжить настройку Nginx на сервере:

```bash
# На сервере
cd ~/Ispravleno/ispravleno-website/website

# Убедитесь, что код обновлен
git pull origin main

# Запустите скрипт настройки
chmod +x setup-ispravleno-pro.sh
./setup-ispravleno-pro.sh
```

## Скриншоты интерфейса reg.ru (примерное расположение)

1. **Личный кабинет → Домены → ispravleno.pro**
2. **Раздел "DNS-серверы и зона"** или **"Управление DNS"**
3. **Подраздел "Записи зоны"** или **"A-записи"**
4. Найдите записи с типом "A" и значениями типа `server263.hosting.reg.ru` или IP адресами серверов reg.ru
5. Удалите или измените их на `212.74.227.208`

## Проверка после изменений

### Через командную строку:

```bash
# Проверка DNS записи
dig ispravleno.pro +short
# Должно вернуть: 212.74.227.208

# Или
nslookup ispravleno.pro
# Должно показать: 212.74.227.208
```

### Онлайн проверка:

- https://dnschecker.org/#A/ispravleno.pro
- https://mxtoolbox.com/DNSLookup.aspx

### Когда DNS обновится:

Откройте в браузере `http://ispravleno.pro` - должна открыться страница вашего сайта (или ошибка подключения, если Nginx еще не настроен, но не страница ошибки reg.ru).

## Важно

⚠️ **Не удаляйте другие записи DNS** (MX, TXT, CNAME и т.д.), если они вам нужны. Меняйте только A-записи, которые указывают на хостинг reg.ru.

Если используете почту на reg.ru или другие сервисы, которые требуют специальных записей - сохраните их.
