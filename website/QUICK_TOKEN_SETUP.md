# Быстрая настройка токена

## Шаг 1: Добавить токен в .env

### Вариант A: Использовать скрипт (рекомендуется)

```powershell
.\setup-token.ps1 -Token "ваш_токен_здесь"
```

### Вариант B: Вручную

1. Откройте файл `.env` в папке `Z:\SeoСайтИсправно`
2. Добавьте или обновите строку:
```env
ADMIN_USER_TOKEN="ваш_токен_здесь"
```

Также убедитесь, что есть:
```env
NEXT_PUBLIC_ADMIN_API_URL="http://212.74.227.208:3000/api"
```

## Шаг 2: Перезапустить сервер

Остановите текущий сервер (Ctrl+C) и запустите снова:
```powershell
npm run dev
```

## Шаг 3: Протестировать

1. Откройте http://localhost:3000/order
2. Создайте тестовый заказ
3. Проверьте логи сервера - должно быть сообщение об успешной отправке
4. Откройте админ-панель → Заказы - должен появиться новый заказ

## Проверка работы

После перезапуска сервера создайте заказ и проверьте:

1. **Логи сервера Next.js** - должно быть:
   ```
   Order sent to admin panel successfully: {...}
   ```

2. **Админ-панель** - раздел "Заказы" должен показать новый заказ

3. **Локальная БД** - заказ должен быть сохранен в таблице `Order`

## Если не работает

1. Проверьте, что токен правильно добавлен в `.env` (без лишних пробелов)
2. Проверьте, что сервер перезапущен после добавления токена
3. Проверьте логи сервера на наличие ошибок
4. Убедитесь, что backend доступен по адресу `NEXT_PUBLIC_ADMIN_API_URL`
