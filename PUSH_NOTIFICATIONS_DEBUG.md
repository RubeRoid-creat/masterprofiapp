# Отладка Push-уведомлений

## Проблема: Push-уведомления не приходят

### Диагностика:

1. **Проверьте наличие FCM токенов в БД:**
   ```bash
   cd backend
   node -e "import('./database/db.js').then(({initDatabase, query}) => { initDatabase().then(() => { const tokens = query.all('SELECT user_id, token FROM fcm_tokens'); console.log('Токенов:', tokens.length); tokens.forEach(t => console.log('User:', t.user_id)); }); })"
   ```

2. **Проверьте логи Android приложения:**
   - Откройте Logcat в Android Studio
   - Фильтр: `FcmService` или `AuthViewModel`
   - Ищите сообщения:
     - "FCM токен получен"
     - "FCM токен успешно зарегистрирован на сервере"
     - Ошибки регистрации

3. **Проверьте логи сервера:**
   - При входе пользователя должно появиться: `✅ FCM токен зарегистрирован для пользователя #X`
   - При изменении статуса заказа должно появиться: `📱 Push-уведомление отправлено пользователю #X`

### Возможные причины:

1. **FCM токен не регистрируется:**
   - Проверьте, что `google-services.json` находится в `ClientApp/app/`
   - Проверьте, что Firebase проект настроен правильно
   - Проверьте логи Android приложения на ошибки

2. **Токен регистрируется, но уведомления не приходят:**
   - Проверьте, что Firebase Admin SDK инициализирован (должно быть сообщение при запуске сервера)
   - Проверьте логи сервера при изменении статуса заказа
   - Убедитесь, что пользователь авторизован и токен сохранен в БД

3. **Уведомления отправляются, но не доходят:**
   - Проверьте, что приложение имеет разрешения на уведомления
   - Проверьте, что устройство подключено к интернету
   - Проверьте, что приложение не находится в фоне (для тестирования)

### Шаги для проверки:

1. **Переустановите приложение** (чтобы получить новый FCM токен)
2. **Войдите в систему** (токен должен зарегистрироваться)
3. **Проверьте БД** - должен появиться токен
4. **Измените статус заказа** - должно отправиться уведомление
5. **Проверьте логи** на обеих сторонах (клиент и сервер)

### Тестирование вручную:

Можно протестировать отправку уведомления вручную через API:
```bash
# Получите токен пользователя
curl -X GET http://localhost:3000/api/fcm/tokens \
  -H "Authorization: Bearer YOUR_TOKEN"

# Отправьте тестовое уведомление (если добавите endpoint)
```
