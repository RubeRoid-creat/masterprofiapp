# 🔧 ИСПРАВЛЕНИЕ ПАРСИНГА ДАТ И ПОЯВЛЕНИЯ ЗАЯВОК

## 🐛 ПРОБЛЕМА

Из логов приложения видно:
- API возвращает назначения (16 назначений)
- Все назначения имеют статус `expired` в поле `expiresAt`
- После фильтрации остается **0 активных назначений**
- Заявки не появляются в приложении

## ✅ ЧТО ИСПРАВЛЕНО

### 1. Парсинг даты с учетом UTC
**Файл:** `app/src/main/java/com/example/bestapp/ui/orders/OrdersViewModel.kt`

Проблема была в том, что `SimpleDateFormat` без указания `TimeZone` неправильно парсил ISO-8601 формат с 'Z' (UTC).

**Исправление:**
```kotlin
// Теперь правильно указываем TimeZone.UTC для форматов с 'Z'
java.text.SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'", java.util.Locale.US).apply {
    timeZone = java.util.TimeZone.getTimeZone("UTC")
}
```

### 2. Нормализация формата даты в API
**Файл:** `backend/routes/assignments.js`

API теперь гарантирует ISO-8601 формат для `expires_at`:
```javascript
if (filtered && filtered.expires_at) {
    const date = new Date(filtered.expires_at);
    if (!isNaN(date.getTime())) {
        filtered.expires_at = date.toISOString();
    }
}
```

### 3. Обработка отсутствующего expiresAt
**Файл:** `app/src/main/java/com/example/bestapp/api/models/ApiModels.kt`

Поле `expiresAt` теперь nullable для совместимости:
```kotlin
@SerializedName("expires_at") val expiresAt: String? // Может быть null
```

### 4. Улучшенное логирование
Добавлены детальные логи:
- Парсинг каждого `expiresAt`
- Сравнение времени (сколько минут осталось/прошло)
- Предупреждения о проблемах

## 📱 ЧТО ПРОВЕРИТЬ

1. **Обновите приложение** до последней версии
2. **Откройте Logcat** и ищите:
   ```
   Парсинг expiresAt: '2025-12-08T16:28:27.608Z' -> ...
   ✅ Назначение #36 активно, осталось X минут
   ```
3. **Если видите ошибки парсинга:**
   ```
   ❌ Не удалось распарсить expiresAt: '...'
   ```
   - Это означает проблему с форматом даты

## 🔍 ДИАГНОСТИКА

Если заявки все еще не появляются:

1. **Проверьте логи приложения:**
   - Ищите `Парсинг expiresAt` - показывает, как парсится дата
   - Ищите `✅ Назначение #X активно` - показывает активные назначения
   - Ищите `⏰ Назначение #X истекло` - показывает истекшие

2. **Проверьте формат даты от API:**
   ```bash
   cd backend
   node scripts/verify-api-response.js
   ```
   Должно показать ISO-8601 формат: `2025-12-08T16:28:27.608Z`

3. **Проверьте текущее время:**
   - Локальное время устройства должно быть правильным
   - Проблемы с часовым поясом могут влиять на сравнение
