# 🔍 Диагностика подключения к серверу

Если приложение не может подключиться к серверу, выполните следующие проверки:

## 1. Проверка запуска сервера

### На сервере выполните:

```bash
# Проверьте, запущен ли сервер через PM2
pm2 status

# Проверьте логи
pm2 logs bestapp-backend --lines 50

# Если сервер не запущен, запустите его
cd /path/to/backend
pm2 start server.js --name bestapp-backend
pm2 save
```

### Проверка порта:

Если `netstat` не установлен, используйте альтернативные команды:

```bash
# Вариант 1: Используйте ss (обычно установлен по умолчанию)
sudo ss -tulpn | grep 3000

# Вариант 2: Используйте lsof (если установлен)
sudo lsof -i :3000

# Вариант 3: Через /proc/net (всегда доступно)
cat /proc/net/tcp | grep 0BB8  # 0BB8 = 3000 в hex

# Вариант 4: Установите netstat (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install net-tools
# Затем:
sudo netstat -tulpn | grep 3000
```

**Ожидаемый результат:** Должен быть процесс Node.js, слушающий на `0.0.0.0:3000` или `*:3000`

## 2. Проверка файрвола

### Ubuntu/Debian (UFW):

```bash
# Проверьте статус файрвола
sudo ufw status

# Если порт 3000 не разрешен, добавьте его
sudo ufw allow 3000/tcp
sudo ufw reload
```

### CentOS/RHEL (firewalld):

```bash
# Проверьте статус
sudo firewall-cmd --list-all

# Добавьте порт, если нужно
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

## 3. Проверка доступности с сервера

### С сервера выполните:

```bash
# Проверьте локальный доступ
curl http://localhost:3000/api/version/check

# Проверьте доступность через внешний IP
curl http://212.74.227.208:3000/api/version/check

# Проверьте, слушает ли сервер на всех интерфейсах
netstat -tulpn | grep 3000
```

**Важно:** Сервер должен слушать на `0.0.0.0:3000`, а не только на `127.0.0.1:3000`

## 4. Проверка конфигурации сервера

Убедитесь, что в `backend/server.js`:

```javascript
server.listen(config.port, '0.0.0.0', () => {
  // ...
});
```

**НЕ должно быть:**
```javascript
server.listen(config.port, 'localhost', () => {
  // Это неправильно для внешнего доступа
});
```

## 5. Использование скрипта диагностики

Запустите скрипт проверки:

```bash
cd backend
node scripts/check-server-connection.js
```

Скрипт проверит:
- ✅ TCP подключение к порту
- ✅ HTTP ответ сервера
- ✅ Доступность API эндпоинта

## 6. Проверка с другого устройства

Попробуйте подключиться с другого компьютера или телефона:

```bash
# С другого устройства в той же сети или интернете
curl http://212.74.227.208:3000/api/version/check
```

## 7. Типичные проблемы и решения

### Проблема: `ECONNREFUSED` (Connection refused)

**Возможные причины:**
1. Сервер не запущен
   - **Решение:** `pm2 start server.js --name bestapp-backend`

2. Сервер слушает только на localhost
   - **Решение:** Проверьте `server.listen()` - должно быть `0.0.0.0`

3. Файрвол блокирует порт
   - **Решение:** `sudo ufw allow 3000/tcp`

4. Сервер на другом порту
   - **Решение:** Проверьте переменную окружения `PORT` или `config.port`

### Проблема: `ETIMEDOUT` (Timeout)

**Возможные причины:**
1. Файрвол блокирует подключение
   - **Решение:** Откройте порт в файрволе

2. Проблемы с сетью
   - **Решение:** Проверьте маршрутизацию и доступность сервера

### Проблема: `EHOSTUNREACH` (Host unreachable)

**Возможные причины:**
1. Неправильный IP адрес
   - **Решение:** Проверьте IP сервера: `hostname -I` или `ip addr`

2. Сервер в другой сети
   - **Решение:** Настройте маршрутизацию или VPN

## 8. Быстрая проверка

Выполните все команды подряд:

```bash
# 1. Проверка статуса PM2
pm2 status

# 2. Проверка порта
sudo netstat -tulpn | grep 3000

# 3. Проверка файрвола
sudo ufw status

# 4. Локальный тест
curl http://localhost:3000/api/version/check

# 5. Внешний тест
curl http://212.74.227.208:3000/api/version/check

# 6. Проверка логов
pm2 logs bestapp-backend --lines 20
```

## 9. Если ничего не помогло

1. Перезапустите сервер:
   ```bash
   pm2 restart bestapp-backend
   pm2 logs bestapp-backend
   ```

2. Проверьте все логи на наличие ошибок:
   ```bash
   pm2 logs bestapp-backend --err
   ```

3. Убедитесь, что сервер запущен и слушает на правильном интерфейсе:
   ```bash
   ps aux | grep node
   sudo netstat -tulpn | grep node
   ```

4. Проверьте переменные окружения:
   ```bash
   cd backend
   cat .env | grep PORT
   ```
