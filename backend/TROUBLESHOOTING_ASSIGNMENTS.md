## 🔍 ВОЗМОЖНЫЕ ПРИЧИНЫ И РЕШЕНИЯ

### Причина 1: Мастер не верифицирован
**Симптомы:**
- В логах диагностики: `❌ pending/not_verified`
- API `/api/assignments/my` возвращает 403

**Решение:**
1. Войдите в админ-панель: http://212.74.227.208:3000/admin
2. Найдите мастера
3. Измените статус верификации на `verified`

**Или через SQL:**
```bash
cd ~/masterprofiapp/backend
sqlite3 database/database.sqlite
```
```sql
-- Проверить мастеров
SELECT id, user_id, verification_status FROM masters;

-- Верифицировать мастера
UPDATE masters SET verification_status = 'verified' WHERE user_id = <ID>;
.quit
```

### Причина 2: Мастер не на смене
**Симптомы:**
- В диагностике: `❌ Не на смене`
- `is_on_shift = 0`

**Решение:**
1. Откройте приложение мастера
2. На экране "Заказы" нажмите кнопку **"Начать смену"**

### Причина 3: Нет подходящей специализации
**Симптомы:**
- В диагностике: `Специализации: НЕТ` или не подходят к типу заказа
- Заказ создан, но assignment не создается

**Решение:**
1. Откройте приложение мастера
2. Перейдите в **Профиль**
3. Нажмите на раздел **"Специализации"**
4. Выберите типы техники (например: "Холодильник", "Стиральная машина")

### Причина 4: Сервер не обновлен
**Симптомы:**
- Старая версия кода на сервере
- `git log` показывает старые коммиты

**Решение:**
```bash
cd ~/masterprofiapp/backend
git fetch origin
git reset --hard origin/main
npm install
pm2 restart all
```

### Причина 5: Assignment не создается при создании заказа
**Симптомы:**
- Заказ есть, но assignments пусто
- В логах нет сообщений от `notifyMasters`

**Решение - проверить логи:**
```bash
pm2 logs server --lines 100
```

Должны быть сообщения:
```
🚀 Запуск процесса назначения для заказа #X
✅ Выбран мастер #Y
✅ Создано назначение #Z
```

Если их нет:
1. Проверьте, что мастер подходит (смена + верификация + специализация)
2. Перезапустите сервер: `pm2 restart all`

## 🧪 ТЕСТИРОВАНИЕ ПОСЛЕ ИСПРАВЛЕНИЯ

### Шаг 1: Проверьте мастера
```bash
node scripts/diagnose-assignments.js
```

Должно быть:
- ✅ НА СМЕНЕ
- ✅ Верифицирован
- Специализации: [список типов техники]

### Шаг 2: Создайте тестовый заказ
1. Откройте **клиентское приложение**
2. Создайте заказ на тип техники из специализации мастера
3. Подождите 2-3 секунды

### Шаг 3: Проверьте назначение
```bash
node scripts/diagnose-assignments.js
```

Должно появиться:
```
📋 Pending: 1
Назначение #X
   Заказ: #Y (Холодильник, new)
   Мастер: #Z - Имя (user_id=...)
   Статус: pending
```

### Шаг 4: Проверьте приложение мастера
1. Откройте приложение мастера
2. Перейдите на экран **"Заказы"**
3. Должен появиться заказ с базовой информацией:
   - Адрес
   - Тип техники
   - Описание проблемы
   - Таймер обратного отсчета (5 минут)

4. Нажмите на заказ
5. Должны быть видны кнопки **"Принять"** и **"Отклонить"**
